# 装箱单自动化工具

一个纯前端 FBA 装箱单转换工具，适合部署到 GitHub Pages。

## 功能

- 批量上传 `.xlsx/.xls` FBA 装箱单。
- 自动识别“每箱一款 SKU”和“每箱多款 SKU”格式，图片列不会影响解析。
- 输出列为：标记号码、货物名称及规格、总箱数、总数量、总毛重、净重。
- 支持折叠查看、复制表格、下载 Excel。多个货件会写入同一个 Excel 的不同 sheet，sheet 名为 FBA 货件号。
- 支持维护 MSKU 档案：货物名称及规格、单件毛重。
- 档案默认保存在浏览器本地，也可导入/导出 JSON 或 Excel；Chrome/Edge 支持连接本地 JSON 档案文件。
- 混装箱支持合并单元格处理、合并名称处理，并提供箱子装箱示意图。
- 支持设置合并报关组，多个 FBA 货件可作为一个输出组展示和导出，名称为 `FBAXXX&FBAXXX的合并报关`。
- 名称/毛重编辑只先应用到当前结果；自动写入 MSKU 档案发生在成功下载装箱单之后。

## 使用

直接打开 `index.html`，或把项目发布到 GitHub Pages。

## 说明

浏览器页面无法任意读写用户电脑上的固定路径，这是网页安全限制。因此档案默认使用浏览器本地存储，并提供 JSON/Excel 导入导出；支持 File System Access API 的浏览器可以手动选择一个 JSON 文件作为档案文件。

浏览器本地存储绑定当前浏览器、当前域名或本地地址。清理浏览器数据、换浏览器、换网址、隐私模式关闭、浏览器策略清空站点数据，都会导致默认档案不可用。重要档案建议定期导出 Excel。
